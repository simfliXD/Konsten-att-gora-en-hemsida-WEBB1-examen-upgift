CSS

Vad är CSS?

CSS (Cascading Style Sheets) är ett stilspråk som används för att bestämma utseendet och layouten på HTML-dokument. Med CSS bestämmer du hur olika HTML-element ska se ut, exempvis vilken färg text ska ha, storleken, eller hur en sida ska bete sig på vid olika skärm storlekar.

"Cascading" syftar på hur regler tillämpas i en viss ordning (kaskad) där specifika regler vinner över generella, och senare regler kan skriva över tidigare regler på samma element.

Syntax

Hur CSS är uppbyggt

En CSS-regel består av en selektor och ett deklarationsblock. Deklarationsblocket innehåller en eller flera egenskaper med värden: 

selektor {
  egenskap: värde;
  egenskap: värde;
}

Tre sätt att skriva CSS

Det finns tre sätt att koppla CSS till ett HTML-dokument:

  1. Extern fil (rekommenderat) – CSS skrivs i en separat .css-fil som länkas in i HTML-filen: 

    2. <link rel="stylesheet" href="style.css">

  2. Intern stil – CSS skrivs direkt i <head> med <style>-taggen: 
    3. <style>
  body {
    background-color: lightblue;
  }
</style>
  3. Inline-stil – CSS skrivs direkt i HTML-elementets style-attribut: 
    4. <p style="color: red; font-size: 48px;">Stor röd text</p>

Selektorer

Selektorer bestämmer vilka HTML-element som CSS-reglerna ska påverka.

  • Tagg-selektor – påverkar alla element av en viss typ: 
    p {
  line-height: 1.6;
}
  • Klass-selektor – påverkar alla element med en viss klass (punktnotation): 
    .highlight {
  background-color: yellow;
}
  • ID-selektor – påverkar ett enskilt element med ett visst ID (brädgårdsnotation): 
    #main-header {
  font-size: 3rem;
}
  • Kombinations-selektor – påverkar element som uppfyller flera villkor: 
    /* Alla <p> inuti en <article> */
article p {
  color: #333;
}

/* Alla element med klassen "card" som också är en <div> */
div.card {
  border: 1px solid black;
}
  • Pseudoklasser – stilar element i ett visst tillstånd, t.ex. vid hovring: 
    a:hover {
  color: orange;
  text-decoration: underline;
}

button:focus {
  outline: 3px solid blue;
}

Viktiga CSS-egenskaper

Färger och bakgrunder

  • color – textfärg. Kan anges som namn, hex, rgb eller hsl: 
    p {
  color: #333333;          /* Hex */
  color: rgb(51, 51, 51);  /* RGB */
  color: hsl(0, 0%, 20%);  /* HSL */
}
  • background-color – bakgrundsfärg på ett element
  • background-image – bakgrundsbild: 
    div {
  background-image: url("bakgrund.jpg");
  background-size: cover;
  background-position: center;
}

HEX-färger är ett vanligt sätt att ange färger i CSS. En HEX-färg börjar med # följt av sex hexadecimala tecken (0-9 och A-F). De första två tecknen representerar röd, de nästa två grön och de sista två blå (RRGGBB). Till exempel:

  • #FF0000 – röd (max röd, ingen grön eller blå)
  • #00FF00 – grön
  • #0000FF – blå
  • #FFFFFF – vit (max alla färger)
  • #000000 – svart (ingen färg)

Förkortad HEX kan användas när varje färgkomponent har samma värde, t.ex. #RGB istället för #RRGGBB. Exempel: #F00 är samma som #FF0000.

Box-modellen

Alla HTML-element är en "låda" bestående av innehåll, padding, border och margin: 

div {
  width: 300px;
  padding: 20px;          /* Inre avstånd */
  border: 2px solid black; /* Ram */
  margin: 10px auto;      /* Yttre avstånd, auto centrerar horisontellt */
}

Text och typografi

  • font-family – typsnittsfamilj
  • font-size – teckenstorlek
  • font-weight – tjocklek (normal, bold, 100–900)
  • line-height – radavstånd
  • text-align – justering (left, center, right)
  • text-decoration – underline, line-through, none
body {
  font-family: "Segoe UI", Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  color: #1d1e22;
}

Layout med Flexbox

Flexbox gör det enkelt att placera element bredvid varandra eller centrera dem: 

.container {
  display: flex;
  justify-content: center;  /* Horisontell placering */
  align-items: center;      /* Vertikal placering */
  gap: 1rem;                /* Avstånd mellan barn-element */
  flex-wrap: wrap;          /* Tillåt radbrytning */
}

Layout med Grid

CSS Grid används för att skapa mer komplexa rutnätslayouter:

.grid {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* Tre kolumner */
  gap: 20px;
}

Varför separera HTML och CSS?

Det finns starka skäl till att hålla HTML och CSS i separata filer:

  • Separation av ansvar – HTML beskriver vad innehållet är, CSS beskriver hur det ser ut. Det gör koden lättare att förstå och underhålla.
  • Återanvändning – en enda CSS-fil kan styra utseendet på hundratals HTML-sidor. Vill du ändra en färg gör du det på ett ställe.
  • Snabbare laddning – webbläsaren cachar CSS-filen, så den behöver bara laddas ned en gång per besökare.
  • Teamwork – designers kan jobba med CSS medan utvecklare jobbar med HTML, utan att skriva över varandras arbete.
  • Enklare testning – du kan byta ut hela utseendet utan att ändra ett enda HTML-element.

Jämför de två varianterna nedan. De ger exakt samma resultat men den externa CSS-varianten är betydligt renare:

Dåligt (inline CSS):

<h1 style="color: orange; font-size: 2rem; text-align: center; margin-bottom: 1rem;">
  Välkommen
</h1>
<p style="color: #333; font-size: 1rem; line-height: 1.6;">
  Lite text här.
</p>

Bra (extern CSS-fil):

/* style.css */
h1 {
  color: orange;
  font-size: 2rem;
  text-align: center;
  margin-bottom: 1rem;
}
p {
  color: #333;
  font-size: 1rem;
  line-height: 1.6;
}
<!-- index.html -->
<h1>Välkommen</h1>
<p>Lite text här.</p>

CSS-variabler

CSS-variabler (officiellt kallade "custom properties") låter dig spara värden som du kan återanvända i hela stilmallen. De definieras med -- och används med var(): 

/* Definiera i :root för att göra dem globala */
:root {
  --primary-color: #ff5a09;
  --text-color: #1d1e22;
  --font-size-base: 1rem;
  --border-radius: 8px;
}

/* Använd dem sedan var som helst */
button {
  background-color: var(--primary-color);
  color: white;
  border-radius: var(--border-radius);
  font-size: var(--font-size-base);
}

h1 {
  color: var(--primary-color);
}

Det stora fördelen är att du kan ändra färgen på hela sajten genom att byta ett enda värde i :root. Det är också hur mörkt läge implementeras på den här sidan – JavaScript lägger till klassen dark-mode på <body> och CSS-variablerna skrivs över med mörkare färger.

Responsiv design och media queries

En responsiv webbsida anpassar sig automatiskt till skärmens storlek – oavsett om den visas på en stor datorskärm eller en liten mobiltelefon.

Viktiga grunder

Alltid börja med denna meta-tagg i <head> – utan den fungerar inte responsiv design på mobila enheter: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Den talar om för mobila webbläsare att använda enhetens faktiska bredd istället för att zooma ut och visa en skrivbordsvy.

Media queries

Med @media kan du skriva CSS-regler som bara aktiveras vid en viss skärmbredd. Det är grunden för att anpassa layouten till mobiler och surfplattor. 

/* Dessa regler gäller alltid (desktop-first) */
h1 {
  font-size: 3rem;
}

/* Dessa regler aktiveras BARA om skärmen är smalare än 1000px */
@media (width < 1000px) {
  h1 {
    font-size: 3vw; /* vw = procent av skärmbredden */
  }
}

/* Ännu smalare – t.ex. mobiltelefon */
@media (width < 600px) {
  h1 {
    font-size: 6vw;
  }

  /* Stapla kolumner vertikalt på mobil */
  .grid {
    grid-template-columns: 1fr;
  }
}

I exemplet ovan används 3vw som enhet. vw står för viewport width – alltså en procent av skärmbredden. Det gör att texten automatiskt skalas med skärmens storlek.

Responsivt konkret exempel

På den här webbsidan används media queries för att stapla sidomenyn och artikeln samt göra navigationen till en hamburgermeny på smala skärmar: 

/* Desktop: sidomeny och artikel sida vid sida */
main {
  display: grid;
  grid-template-columns: auto 1fr;
}

/* Mobil: stapla artikel och sidomeny vertikalt, sidomeny full bredd */
@media screen and (max-width: 750px) {
  main {
    grid-template-columns: auto; /* En ända kolumn */
  }

  aside {
    position: static; /* Inte längre sticky utan den stannar på sin plats */
    width: 100%;
  }
}

Responsiv design utan media queries

Det går faktiskt att göra en helt responsiv design utan media queries – genom att använda flexbox eller grid tillsammans med flexibla enheter som fr, vw, vh, %, em och rem. Det kräver mer planering och kreativitet

Det enklast sättet att hålla en sida resonsiv utan media queries är med clamp

clamp(min, ideal, max) – håller ett värde inom ett intervall, perfekt för responsiv typografi: 
h1 {
  /* Aldrig mindre än 1.5rem, aldrig större än 3rem */
  font-size: clamp(1.5rem, 4vw, 3rem);
}

Bra enheter att känna till

  • px – pixlar, fast storlek
  • % – procent av förälderns storlek
  • rem – relativt till rot-elementets teckensnittstorlek
  • em – relativt till det aktuella elementets teckensnittstorlek
  • vw – procent av viewport bredden
  • vh – procent av viewport höjden

Prova själv – ändra sidans färger

CSS-variabler gör det möjligt att ändra hela sidans utseende med bara några rader JavaScript. Välj bakgrunds- och textfärg nedan och tryck på Aktivera – förändringen sker direkt på den här sidan, ovanpå både ljust och mörkt läge.

Tekniken fungerar genom att JavaScript skapar ett nytt <style>-element i <head> med regler som matchar :root, body, body.dark-mode. Eftersom dessa regler läggs till sist i dokumentet vinner de i kaskaden och skriver över befintliga variabler.
