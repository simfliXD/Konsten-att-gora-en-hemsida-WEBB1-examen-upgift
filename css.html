CSS

Vad är CSS?

CSS (Cascading Style Sheets) är ett stilspråk som används för att bestämma utseendet och layouten på HTML-dokument. Med CSS bestämmer du hur olika HTML-element ska se ut, exempvis vilken färg text ska ha, storleken, eller hur en sida ska bete sig på vid olika skärm storlekar.

"Cascading" syftar på hur regler tillämpas i en viss ordning (kaskad) där specifika regler vinner över generella, och senare regler kan skriva över tidigare regler på samma element.

Tre sätt att skriva CSS

Det finns tre sätt att koppla CSS till ett HTML-dokument:

  1. Extern fil (rekommenderas) – CSS skrivs i en separat .css-fil som länkas in i HTML-filen: 
    <link rel="stylesheet" href="style.css">
  2. Intern stil – CSS skrivs direkt i <head> med <style>-taggen: 
    <style>
  body {
    background-color: lightblue;
  }
</style>
  3. Inline-stil – CSS skrivs direkt i HTML-elementets style-attribut: 
    <p style="color: red; font-size: 48px;">Stor röd text</p>

Varför separera HTML och CSS?

Det finns starka skäl till att hålla HTML och CSS i separata filer:

  • Separation av ansvar – HTML beskriver vad innehållet är, CSS beskriver hur det ser ut. Det gör koden lättare att förstå och underhålla.
  • Återanvändning – en enda CSS-fil kan styra utseendet på hundratals HTML-sidor. Vill du ändra en färg gör du det på ett ställe.
  • Snabbare laddning – webbläsaren cachar CSS-filen, så den behöver bara laddas ned en gång per besökare.
  • Teamwork – designers kan jobba med CSS medan utvecklare jobbar med HTML, utan att skriva över varandras arbete.
  • Enklare testning – du kan byta ut hela utseendet utan att ändra ett enda HTML-element.

Jämför de två varianterna nedan. De ger exakt samma resultat men den externa CSS-varianten är betydligt renare:

CSS-syntax

Hur CSS är uppbyggt

En CSS-regel består av en selektor och ett deklarationsblock. Deklarationsblocket innehåller en eller flera egenskaper med värden: 

selektor {
  egenskap: värde;
  egenskap: värde;
}

Selektorer

Selektorer bestämmer vilka HTML-element som CSS-reglerna ska påverka.

  • Tagg-selektor – påverkar alla element av en viss typ: 
    p {
  line-height: 1.6;
}
  • Klass-selektor – påverkar alla element med en viss klass: 
    .highlight {
  background-color: yellow;
}
  • ID-selektor – påverkar ett enskilt element med ett visst ID: 
    #main-header {
  font-size: 3rem;
}
  • Kombinations-selektor – påverkar element som uppfyller flera villkor: 
    /* Alla <p> inuti en <article> */
article p {
  color: #333;
}

/* Alla element med klassen "card" som också är en <div> */
div.card {
  border: 1px solid black;
}
  • Pseudoklasser – stilar element i ett visst tillstånd, te.x när ett element är i focus : 
    a:hover {
  color: orange;
  text-decoration: underline;
}

button:focus {
  outline: 3px solid blue;
}

Pseudo-element

Pseudo-element börjar med : och skapar virtuella element som inte finns i HTML-strukturen.

:before och :after

Skapar innehåll före/efter ett elements faktiska innehåll. Används ofta för dekorativa element eller clearfix: 

.icon:before {
  content: "→ ";
}

.box:after {
  content: " - klar";
  color: green;
}

:first-letter

Påverkar första bokstaven i ett element:

p:first-letter {
  font-size: 2em;
  font-weight: bold;
}

:first-line

Påverkar första raden i ett element:

p:first-line {
  color: blue;
}

:selection

Påverkar markerad text:

:selection {
  background: yellow;
  color: black;
}

:nth-child(n)

Påverkar det n:te barnet till en förälder:

li:nth-child(2) {
  font-weight: bold;
}

CSS-funktioner

CSS har inbyggda funktioner som kan användas som värden i egenskaper.

url()

Laddar externa resurser som bilder:

div {
  background-image: url("https://example.com/bild.jpg");
}

calc()

Beräkningar med kombination av enheter:

div {
  width: calc(100% - 20px);
  margin: calc(10px + 5px);
}

min(), max(),minmax(), clamp()

Begränsa värden inom intervall, bra för responsiv design:

width: min(50%, 300px);   /* Välj det mindre */
width: max(200px, 20%);   /* Välj det större */
width: minmax(0, 1fr);   /* Max 1fr, annars 0 */
width: clamp(150px, 50%, 400px); /* Inom intervall (min, preferred, max) */

var()

CSS-variabler:

:root {
  --primary-color: #3498db;
  --spacing: 16px;
}
div {
  color: var(--primary-color);
  padding: var(--spacing);
}

counter()

Skapar räknare för numrering:

body {
  counter-reset: section;
}
h2::before {
  counter-increment: section;
  content: "Section " counter(section) ": ";
}

Box-modellen

Alla HTML-element är en "låda" bestående av innehåll, padding, border och margin: 

div {
  width: 300px;
  padding: 20px;          /* Inre avstånd */
  border: 2px solid black; /* Ram */
  margin: 10px auto;      /* Yttre avstånd, auto centrerar horisontellt */
}

box-sizing

box-sizing bestämmer hur width och height beräknas. Standard är content-box (endast innehåll). Med border-box inkluderas padding och border i måttet: 

/* Inkluderar padding och border i width */
div {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid black;
}

Positionering

Med position kan du styra hur element placeras i förhållande till normala flödet eller andra element.

position: static

Standardvärde. Elementet flödar normalt i dokumentet.

position: relative

Elementet placeras relativt till sin normala position. Du kan förskjuta det med top, right, bottom, left utan att påverka andra element: 

.box {
  position: relative;
  top: 20px;
  left: 10px;
}

position: absolute

Elementet tas bort från normala flödet och placeras relativt till närmaste positionerade förälder (eller viewport): 

.overlay {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}

position: fixed

Elementet positioneras relativt till viewporten och stannar på plats även vid scrollning: 

.back-to-top {
  position: fixed;
  bottom: 20px;
  right: 20px;
}

position: sticky

Elementet beter sig som relative tills det når en specifik position, därefter "klibbar" det fast: 

.sticky-header {
  position: sticky;
  top: 0;
}

z-index

z-index kontrollerar stapelordningen. Högre värden hamlar framför lägre: 

.modal {
  position: fixed;
  z-index: 1000;
}

Färger och bakgrunder

  • color – textfärg. Kan anges som namn, hex, rgb eller hsl: 
    p {
  color: #333333;          /* Hex */
  color: rgb(51, 51, 51);  /* RGB */
  color: hsl(0, 0%, 20%);  /* HSL */
}
  • background-color – bakgrundsfärg på ett element
  • background-image – bakgrundsbild: 
    div {
  background-image: url("bakgrund.jpg");
  background-size: cover;
  background-position: center;
}

HEX-färger är ett vanligt sätt att ange färger i CSS. En HEX-färg börjar med # följt av sex hexadecimala tecken (0-9 och A-F). De första två tecknen representerar röd, de nästa två grön och de sista två blå (RRGGBB). Till exempel:

  • #FF0000 – röd (max röd, ingen grön eller blå)
  • #00FF00 – grön
  • #0000FF – blå
  • #FFFFFF – vit (max alla färger)
  • #000000 – svart (ingen färg)

Förkortad HEX kan användas när varje färgkomponent har samma värde, t.ex. #RGB istället för #RRGGBB. Exempel: #F00 är samma som #FF0000.

Text och typografi

  • font-family – typsnittsfamilj
  • font-size – teckenstorlek
  • font-weight – tjocklek (normal, bold, 100–900)
  • line-height – radavstånd
  • text-align – justering (left, center, right)
  • text-decoration – underline, line-through, none
body {
  font-family: "Segoe UI", Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  color: #1d1e22;
}

Layout med Flexbox

Flexbox gör det enkelt att placera element bredvid varandra eller centrera dem: 

.container {
  display: flex;
  justify-content: center;  /* Horisontell placering */
  align-items: center;      /* Vertikal placering */
  gap: 1rem;                /* Avstånd mellan barn-element */
  flex-wrap: wrap;          /* Tillåt radbrytning */
}

Layout med Grid

CSS Grid används för att skapa mer komplexa rutnätslayouter:

.grid {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* Tre kolumner */
  gap: 20px;
}

2D-transformer

Med transform kan du rotera, skala och flytta element utan att påverka dokument strukturen.

translate()

Flyttar element i x- och y-led:

.move-right {
  transform: translate(20px, 10px);
}

/* Separata axlar */
transform: translateX(20px);
transform: translateY(-10px);

rotate()

Rotera element:

.rotate {
  transform: rotate(45deg);
}

scale()

Skala element (förstora/minska):

.bigger {
  transform: scale(1.5);
}

/* Separat för X och Y */
.transform: scale(1, 0.5);  /* Spegelvänd */
transform: scaleX(2);       /* Dubbel bredd */
transform: scaleY(0.5);     /* Halv höjd */

Transitioner och animationer

Transitioner skapar mjuka övergångar mellan tillstånd. Animationer är mer komplexa rörelser.

Transitioner

Med transition animeras egenskapsförändringar: 

.button {
  background-color: blue;
  transition: background-color 0.3s ease;
}

.button:hover {
  background-color: red;
}

Timing-funktioner

  • ease – långsam start, snabb mitt, långsam slut
  • linear – konstant hastighet
  • ease-in – långsam start, snabb slut
  • ease-out – snabb start, långsam slut
  • ease-in-out – långsam start och slut
  • cubic-bezier() – anpassad hastighetskurva

Animationer

Med @keyframes definieras animationssteg:

@keyframes bounce {
  0%, 100% {
    transform: translateY(0);
  }
  50% {
    transform: translateY(-20px);
  }
}

.element {
  animation: bounce 1s infinite;
}

Animationsegenskaper: animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count, animation-direction, animation-fill-mode, animation-play-state.

Exempel med fler egenskaper

.loader {
  animation: spin 2s linear infinite;
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

Responsiv design

En responsiv webbsida anpassar sig automatiskt till skärmens storlek – oavsett om den visas på en stor datorskärm eller en liten mobiltelefon.

Utan viewport meta-taggen fungerar inte responsiv design: 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Den talar om för webbläsaren att använda enhetens faktiska bredd istället för att zooma ut och visa en skrivbordsvy.

Media queries

Med @media kan du skriva CSS-regler som bara aktiveras när vissa kriterier uppfylls. Grunden för att kunna anpassa layouten till mobiler och surfplattor. 

/* Dessa regler gäller alltid */
h1 {
  font-size: 3rem;
}

/* Dessa regler aktiveras BARA om skärmen är smalare än 1000px */
@media (width < 1000px) {
  h1 {
    font-size: 3vw; /* vw = procent av skärmbredden */
  }
}

/* Ännu smalare – t.ex. mobiltelefon */
@media (width < 600px) {
  h1 {
    font-size: 6vw;
  }

  /* Stapla kolumner vertikalt på mobil */
  .grid {
    grid-template-columns: 1fr;
  }
}

I exemplet ovan används 3vw som enhet. vw står för viewport width – alltså procent av skärmbredden. Det gör att texten automatiskt skalas med skärmens storlek.

Responsiv design utan media queries

Det går faktiskt att göra en helt responsiv design utan massor av media queries . Man kan använda flexbox eller grid tillsammans med flexibla enheter som fr, vw, vh, %, em och rem.

Det enklast sättet att hålla en sida resonsiv utan media queries är med clamp

clamp(min, ideal, max) – håller ett värde inom ett intervall, perfekt för responsiv typografi: 
h1 {
  /* Aldrig mindre än 1.5rem, aldrig större än 3rem */
  font-size: clamp(1.5rem, 4vw, 3rem);
}

Bra enheter att känna till

  • px – pixlar, fast storlek
  • % – procent av förälderns storlek
  • rem – relativt till rot-elementets teckensnittstorlek
  • em – relativt till det aktuella elementets teckensnittstorlek
  • vw – procent av viewport bredden
  • vh – procent av viewport höjden

Fler media-egenskaper

Utöver bredd och höjd finns många andra media-egenskaper till exempel:

  • aspect-ratio – skärmens bildförhållande
  • orientation – portrait eller landscape
  • resolution – skärmens pixeldensitet (dpi/dpcm)
  • prefers-color-scheme – mörkt eller ljust läge
  • prefers-reduced-motion – användaren föredrar mindre animation
/* Mörkt läge */
@media (prefers-color-scheme: dark) {
  body { background: #111; color: #fff; }
}

/* Mindre animation */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

Logiska operatorer

Kombinera villkor med and, not, only:

/* Flera villkor */
@media (min-width: 600px) and (max-width: 900px) {
  .grid { columns: 2; }
}

@media not (max-width: 600px) {
  .desktop { display: block; }
}

}

Rangesyntax

Modern syntax för att ange intervall:

/* Istället för: min-width: 400px */
@media (width >= 400px) { }

/* Istället för: min-width: 400px and max-width: 800px */
@media (400px <= width <= 800px) { }

@supports

Testa om webblädaren stöder en CSS-egenskap:

@supports (display: grid) {
  .grid { display: grid; }
}

@supports not (display: grid) {
  .grid { float: left; }  /* Fallback */
}

CSS-variabler

CSS-variabler (officiellt kallade "custom properties") låter dig spara värden som du kan återanvända i hela stilmallen. De definieras med -- och används med var(): 

/* Definiera i :root för att göra dem globala */
:root {
  --primary-color: #ff5a09;
  --text-color: #1d1e22;
  --font-size-base: 1rem;
  --border-radius: 8px;
}

/* Använd dem sedan var som helst */
button {
  background-color: var(--primary-color);
  color: white;
  border-radius: var(--border-radius);
  font-size: var(--font-size-base);
}

h1 {
  color: var(--primary-color);
}

Det stora fördelen är att du kan ändra färgen på hela sajten genom att byta ett enda värde i :root. Det är också hur mörkt läge implementeras på den här sidan – JavaScript lägger till klassen dark-mode på <body> och CSS-variablerna skrivs över med mörkare färger.

Arv (Inheritance)

Vissa CSS-egenskaper ärvs automatiskt från förälder till barn. Exempel: color, font-family, font-size. 

/* Alla <p> ärver färgen från <article> */
article {
  color: blue;
}

p {
  /* Behöver inte ange color, de ärvs automatiskt i en struktur med article -> p */
}

Tvinga arv

Med inherit kan du tvinga arv:

.button {
  color: inherit; /* Ärver textfärg från förälder */
}

Återställ till standard

.reset {
  color: initial;  /* Återställ till standardvärde */
  color: unset;    /* Återställ till ärvda värdet eller standard */
}

Specificitet

När flera regler gäller samma element vinner den med högst specificitet.

  • style="" (inline) = 1000
  • #id = 100
  • .class, :pseudo = 10
  • element, ::pseudo-element = 1
  • * (universell) = 0
/* Specificitet: 10 + 1 = 11 */
p.intro {
  color: blue;
}

/* Specificitet: 1 */
p {
  color: red; /* Förlorar mot .intro */
}

!important

!important åsidosätter specificitetsordning och går över allt. 

.important {
  color: red !important;
}

Anpassad styling – ändra sidans färger

CSS-variabler gör det möjligt att ändra hela sidans utseende med bara några rader JavaScript. Välj bakgrunds- och textfärg nedan och tryck på Aktivera.

Det fungerar genom att med hjälp av JavaScript skapa en ny <style> element i <head> med regler som matchar :root, body, body.dark-mode. Eftersom dessa regler läggs till sist i dokumentet vinner de i kaskaden och skriver över variablarna.

Webbläsarens utvecklarverktyg

Alla moderna webbläsare har inbyggda utvecklarverktyg (DevTools) som underlättar felsökning och experimentering med CSS.

Öppna DevTools

  • Chrome/Edge: F12 eller Ctrl+Shift+I (Cmd+Shift+I på Mac)
  • Firefox: F12 eller Ctrl+Shift+I
  • Safari: Alt+Cmd+I

Inspect Element

Högerklicka på ett element och välj "Inspect" för att se dess HTML och tillämpade CSS. Du kan ändra värden direkt i webbläsaren för att experimentera.

Fliken Styles

  • Visar alla regler som appliceras på elementet
  • Grön text = överskriven (line-through)
  • Klicka på färgrutor för att ändra färger
  • Klicka på box-modellen för att visualisera padding, border, margin

Fliken Computed

Visar slutgiltiga beräknade värden för alla egenskaper.

Tvinga element-tillstånd

I DevTools kan du tvinga :hover, :focus, :active, :visited för att testa stilar utan att interagera med sidan.

Tillgänglighet

Tillgänglig webbdesign (accessible) innebär att alla användare kan uppfatta och interagera med webbplatsen, oavsett funktionsnivå.

Färgkontrast

WCAG kräver tillräcklig kontrast mellan text och bakgrund:

  • Normal text: minst 4.5:1 kontrast
  • Stor text: minst 3:1
  • UI-komponenter: minst 3:1

Relativa enheter

Använd relativa enheter så de skalas:

/* Bra - anpassar sig efter användarens inställningar */
font-size: 1rem;     /* Root-em, baserat på webbläsar inställning */
font-size: 1.2em;    /* Relativt till förälder */

/* Undvik - fast storlek som inte skalas, kommer se dåligt ut för olika skärmar */
font-size: 16px;

Tydlig focus stil

Element som länkar ska ha synliga focus-indikatorer:

button:focus { outline: 3px solid blue; }
button:focus-visible { outline: 3px solid blue; }  /* Endast för tangentbord */
}
